## D-001 (2026-07-04) 기록 시스템은 같은 리포의 context/ 디렉토리

- **결정**: 별도 리포가 아닌 이 리포 안의 `context/`에 모든 과정 기록. 형식은 마크다운 기본.
- **이유**: 코드와 기록의 버전 일치, 커밋 = 코드+기록 동시 스냅샷, 에이전트 접근 마찰 최소화. HTML은 본선 발표 시점에 기록으로부터 생성.
- **버린 대안**: 별도 컨텍스트 리포(버전 어긋남·기록 누락 위험), Notion(에이전트 접근 마찰).

## D-002 (2026-07-04) 제출은 검증 하네스 통과를 강제

- **결정**: 모든 제출물은 `scripts/make_submit.py` 경유 — tests → git clean → 패키징 → 12개 검증 → 자동 기록. 수동 zip 금지.
- **이유**: 일 10회 제출 예산을 형식 오류·시간 초과로 태우는 것 방지. 공식 baseline_submit.zip으로 하네스 검증 완료 (12/12 PASS, 네트워크 차단 네거티브 테스트 확인).
- **버린 대안**: 수기 체크리스트만 운영 (사람은 잊는다).

## D-003 (2026-07-04) CV는 세션 프리픽스 GroupKFold

- **결정**: `id`의 `sess_...` 프리픽스를 그룹 키로 GroupKFold. 로컬 CV로 의사결정, 제출은 검증용.
- **이유**: 같은 세션의 step들이 train/valid에 갈라지면 누수 → CV 과대평가. Private Score 기준이라 public 과적합도 경계.
- **버린 대안**: 무작위 KFold (누수), 시간 기준 split (세션 시각 정보 불충분 — EDA 후 재검토 가능).

## D-004 (2026-07-04) 솔루션 순서: Tier 0 → 1 → 2/3, Tier 2/3는 사실상 필수

- **결정**: 베이스라인 제출(파이프라인 검증) → GBDT+피처 → 경량 인코더 → 앙상블+threshold. 마감 D-3부터 새 모델 금지. (threshold 부분은 → D-009로 대체)
- **이유**: LB가 1등 0.79015 / 12등 0.77585로 극도로 촘촘 — TF-IDF 계열만으로는 커트라인(0.776) 도달 어려움. 단, 빠른 반복이 가능한 Tier 1로 피처 신호를 먼저 확보해야 Tier 2가 산다.
- **버린 대안**: 처음부터 인코더 직행 (파이프라인·CV 신뢰 없이는 실험이 눈먼 상태).

## D-006 (2026-07-04) w112(LB 0.7208)를 출발점으로 채택 — Tier 재구축 로드맵 폐기

- **결정**: 팀의 기존 최고 제출물 w112(3-way 앙상블: linear + AAR stacker + e5-base 인코더, weights [1,1,2])를 공식 출발점으로 채택. PLAN의 "Tier 0부터 재구축" 로드맵은 폐기하고, 개선 방향은 핸드오프 §8 우선순위(시뮬레이터 포렌식 1순위)를 따른다.
- **이유**: Tier 0~3이 이미 다른 작업공간에서 완료·LB 검증됨 (0.43 → 0.7208). 재구축은 시간 낭비. 커트라인까지 −0.055는 미세 앙상블 튜닝으로 불가함이 실측으로 확인됨 (가중 곡선 체감: +0.0010 → +0.0008).
- **계승하는 게이트**: 점수 판정은 LB 실측 또는 세션 group-split만 / per-class F1 항상 확인 / 폐기 목록(experiments.md) 재시도 금지 / 로컬 CV는 가족별 할인율 적용해서 읽기.
- **근거 문서**: [legacy/w112_handoff.md](../legacy/w112_handoff.md)

## D-007 (2026-07-05) 결정적 규칙 노선 기각 — R3(첫스텝 prior)·R4(계층 분류)로 전환

- **결정**: "시뮬레이터 결정 규칙 발견 → 해당 구간 F1 ~1.0" 노선을 포렌식 1라운드 실측으로 기각. 다음 우선순위는 R4(explore 4클래스 계층 분류 프로토타입)와 R3(첫스텝 전용 class-wise prior 보정), 순서대로 로컬 CV → LB 게이트.
- **이유**: 순수 구조 state 조합의 purity≥0.99 규칙 누적 coverage 0.03%(min_rows≥10이면 0.016%, ≥12면 0%) — 규칙화 가능한 결정 구간이 없음. 템플릿 포함 상한 7.4%도 절반(49.7%)이 이미 F1 1.0인 respond_only 몫이라 순증 3.7%뿐이고 과적합 위험. (수치는 독립 리뷰로 재검증됨) 반면 (last2,last1) 조건부 explore purity 80~90%와 첫스텝 분포 이질성(12.9% coverage)은 구조 변경으로만 회수 가능한 신호.
- **버린 대안**: R2 템플릿 오버라이드(소표본 암기 위험, 이득 미미 — 시간 남을 때만 소수 고빈도 템플릿 프로브), R1 respond_only 종료 규칙(이미 포화 + 히든 테스트가 세션당 다중 행인지 미확인).
- **근거 문서**: [reports/forensics_r1.md](reports/forensics_r1.md)

## D-008 (2026-07-05) 세션 형제 행 라벨 복원 패스를 모든 제출물에 내장

- **결정**: script.py에 라벨 복원 후처리를 상시 내장한다 — test.jsonl을 세션 프리픽스로 그룹핑·step 정렬 후, step k+1 행이 존재하면 그 history의 마지막 `assistant_action.name`을 step k의 예측으로 사용(형제 행이 없으면 모델 예측 폴백). 오늘 단독 프로브 1회로 test 구조를 판별한다.
- **이유**: train 실측에서 연속 스텝 쌍 58,326/58,326(100.00%) 성립 — 시뮬레이터가 이전 행동을 다음 스텝 history에 그대로 기록한다(sim·au 계열 공통). 성립 시 test 비최종 스텝 전부가 무료 정답. 폴백 구조라 하방 위험 0, 추가 의존성 0.
- **주의**: 로컬 test 스텁 5행은 전부 다른 세션 → 주최측이 세션당 1스텝만 샘플링했을 가능성이 높다(그 경우 이득 0). 상위권 점수가 0.79대(0.95+가 아님)인 것도 이를 시사. 그래서 "기대 레버"가 아니라 "무료 보험"으로 취급하고, 점수 기대는 R4·인코더 개선에 둔다.
- **근거**: experiments.md #12, 검증 스크립트는 스크래치(재현: 세션 그룹핑 → 연속 쌍 → history[-1].name 대조).

## D-009 (2026-07-06) threshold/prior/calibration 가족 폐기 — Tier 3 마감 레인은 서브모집단 라우팅

- **결정**: D-004 로드맵의 "앙상블+threshold" 중 threshold 튜닝(및 prior 보정·calibration 등 확률 재조정 가족 전체)을 폐기한다. Tier 3의 점수 레버는 "결정적 키로 판별되는 서브모집단 + 전용 모델 라우팅"(soft-AU가 실증)과 이질 인코더 추가로 대체.
- **이유**: calib_v1에서 리그 이득이 LB로 비전이 실측(holdout +0.005 → LB −0.002). 같은 가족인 R3 첫스텝 prior는 이 근거로 미제출 기각(exp #9). 확률 재조정 가족은 홀드아웃 잡음에 맞추는 구조라 private 전이를 기대할 수 없다. 반면 라우팅 축은 리그→LB 전이가 2연속 실측됨(soft-AU: 리그 +0.0065 → LB +0.0069).
- **버린 대안**: 클래스별 threshold 재시도(가족 전체가 동일 실패 메커니즘), 버킷별 가중(exp #20 LB −0.0061 — enc 지분 축 신기루).
- **근거 문서**: experiments.md #9, #19/#20, #24 / PLAN.md §3.1
