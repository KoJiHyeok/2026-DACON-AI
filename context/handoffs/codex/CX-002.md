## Known limitations

- `diagnostic_meta_cv` cross-fits only the meta layer over fixed base OOF. Base models used for meta-train rows may have trained on diagnostic validation groups. This is not end-to-end nested CV; a frozen shadow set excluded from all base/meta fitting or outer-fold base regeneration is required for a promotion-quality estimate.
- The baseline is `legacy_linear_proxy`, not a verified alpha09 sparse OOF. This is a non-parity plumbing/diagnostic run; `teammate_baseline_parity_verified=false`, `teammate_parity=false`, and `promotion_eligible=false` are mandatory.
- The equal 50:50 blend is an explicit local comparison proxy, not the deployed champion blend. No superiority or production-score claim is made against the current champion.
- An NPZ-internal `id`↔`probs` row-order mismatch cannot be detected structurally. Mitigation is generator-side same-traversal saving of IDs/probabilities plus the handoff SHA256 lock; the six supplied hashes passed.
- Final all-OOF fit is not scored in sample. Coefficient contribution magnitude is descriptive, not causal importance.
- No optional mBERT run was attempted; required validation, convergence repair, training, diagnostics, and documentation took priority.
