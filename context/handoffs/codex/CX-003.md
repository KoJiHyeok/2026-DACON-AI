## New hypothesis cards

Each card reports two deliberately separate oracle quantities:

1. **Absolute eligible-set oracle ceiling**: correct every eligible named error with `y_true` and recompute 14-class Macro-F1 exactly. This is a true ceiling only for interventions limited to that eligible set, and it remains oracle/unimplementable.
2. **Rate-conditioned favorable oracle scenario**: correct the judgmentally assumed realistic-rate count, favorably choosing lowest-margin eligible errors, then recompute the metrics. The correction rate is a judgment call, not an estimate from this holdout; the result is a scenario, **not a bound** and not measured candidate performance.

The conservative card-level no-submit threshold uses only the rate-conditioned scenario, never the absolute ceiling. The 9,969-row discovery holdout is restricted to taxonomy and hypothesis generation and is forbidden for card validation. Every card requires either an untouched session-group shadow population or a newly generated annotation-contract population, followed there by all five league metrics: row Macro-F1, session-uniform Macro-F1, one-row/session MC200, paired-session bootstrap2000, and deterministic halves. No candidate was trained or fit.

## Limitations

1. This is a single fixed 9,969-row discovery holdout. It is usable only for taxonomy/hypothesis generation, not validation; descriptive damage, ceilings, and scenarios do not establish private/LB transfer.
2. Oracle corrections know `y_true`; the absolute eligible-set result is a ceiling only within that eligible set, while the judgmental-rate result is merely a favorable scenario. Their five-metric stability measures sampling stability of oracle corrections, not implementability.
3. Component OOF artifacts align exactly by ID but are not exact provenance parity with the champion-building components. No blend reconstruction is valid.
4. Legacy linear hashes are observed only because its root has no manifest.
5. Full observed session length and relative turn are unavailable under the one-row-per-test-session contract.
6. Aggregate regex/workspace lifts are univariate associations with minimum support, not causal or pure rules. No verbatim prompts were retained in outputs.
7. H1/H3 exceed +0.005 only in favorable rate-conditioned scenarios; neither has candidate predictions or independent-population validation, so neither is promotion/submission eligible.
